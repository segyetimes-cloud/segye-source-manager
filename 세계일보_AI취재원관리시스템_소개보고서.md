# 세계일보 AI 기반 취재원 관리시스템
### 도입 소개 보고서

---

## 1. 개요

세계일보 취재원 관리시스템은 기자들이 개인 수첩·엑셀·메모 등에 분산 보관하던 취재원 정보를
**조직 차원의 지식 자산**으로 통합·관리하는 사내 전용 웹 플랫폼입니다.

> 🌐 **접속 주소:** https://segye-source-manager.vercel.app

---

## 2. 왜 만들었나

| 기존 문제 | 이 시스템의 해결책 |
|-----------|----------------|
| 기자 개인 수첩·엑셀에 취재원 정보 분산 | 전사 단일 DB에 통합 |
| 퇴사·이직 시 취재원 정보 소멸 | 조직 자산으로 영구 보존 |
| "저 취재원 누가 알아?" 물어보기 어려움 | 공유 목록 + 도움 요청 게시판 |
| 같은 인물을 여러 기자가 중복 등록 | 중복 감지 자동 경고 |
| 민감한 정보 유출 걱정 | 역할별 접근 제한 + 워터마크 |

## 3. 주요 기능 한눈에 보기

### 3-1. 취재원 등록 및 관리

- **전체 정보 입력**: 이름, 소속, 직책, 연락처, 학력, 고향, 고시기수, SNS 등
- **완성도 점수 자동 계산**: 입력 항목에 따라 0~100점으로 표시, 정보가 많을수록 고득점
- **직책 이력 관리**: 현 직책 외 이전 경력도 타임라인으로 보존
- **수정 이력 추적**: 누가 언제 어떤 내용을 바꿨는지 상세 기록
- **수정 전 확인 절차**: 저장 전 변경 내용 미리보기로 실수 방지

### 3-2. 정보(메모) 공유 시스템

- **기자별 메모 분리 표시**: A기자·B기자·C기자가 각자 입력한 내용을 별도 카드로 구분
- **통합 정보 자동 생성**: 여러 기자의 메모에서 중복 제거 후 하나의 통합 정보로 요약 표시
- **작성자 식별**: 누가 언제 입력한 정보인지 한눈에 확인

### 3-3. 공개 범위 설정 (4단계 보안)

```
내 목록 (나만 열람)
    ↕
편집국 공유 + 공개 (전체 기자 열람)
    ↕
편집국 공유 + 민감 (데스크·슈퍼관리자만 열람)
    ↕
민감 정보 열람 승인 요청 (데스크 승인 필요)
```

### 3-4. AI 기반 엑셀 가져오기

1. 기자가 보유한 엑셀 파일 업로드
2. **Claude AI가 컬럼 자동 분류** (이름/전화/소속 등 자동 매핑)
3. 매핑 결과 미리보기 → 수정 후 확인
4. 일괄 저장 → 포인트 자동 지급

### 3-5. 관계망 그래프

- 취재원 간 인맥·조직 관계를 **시각적 네트워크 지도**로 표시
- 같은 위원회·기수·학교 출신 자동 연결
- 노드 클릭 시 해당 취재원 상세 페이지 바로 이동

### 3-6. 도움 요청 게시판

- "○○부처 실무자 아는 분?"과 같은 취재원 요청 게시
- 응답 → 채택 → 포인트 자동 지급
- 실시간 알림 (Supabase Realtime 기반)

### 3-7. 게이미피케이션 (자발적 참여 유도)

| 행동 | 포인트 |
|------|--------|
| 취재원 등록 (완성도 90점↑) | +30pt |
| 취재원 등록 (60~89점) | +15pt |
| 내 취재원을 타인이 조회 | +2pt |
| 유용성 4~5점 평가 수신 | +10pt |
| 도움 요청 채택 | +20pt |
| 일일 로그인 | +1pt |

- 전체 **리더보드** 공개로 선의의 경쟁 유도
- 내 포인트 내역은 본인만 열람

### 3-8. 관리자 기능

| 기능 | 설명 |
|------|------|
| 계정 관리 | 가입 승인, 역할 변경, 비활성화 |
| 민감 정보 승인 | 열람 요청 승인·거절 |
| 접근 로그 | 누가 언제 어떤 취재원을 열람했는지 전수 기록 |
| 실적 집계 | 기자별 등록 건수·완성도·기여도 엑셀 다운로드 |

## 4. 보안 설계

### 워터마크
- 모든 화면에 **로그인 계정 + 접속 시각이 대각선으로 반복 삽입**
- 육안으로는 거의 보이지 않지만 스마트폰 촬영 시 사진에 선명하게 노출
- 개발자 도구로 제거 시도 시 즉각 재주입 (MutationObserver)
- **화면 무단 촬영·캡처 유출자 특정 가능**

### 역할별 접근 제어 (DB 레벨 RLS)
```
슈퍼관리자  전체 열람·수정·삭제
데스크      민감 정보 열람, 승인 처리, 500행 내보내기
차장        공유 공개 정보 열람, 100행 내보내기
기자        본인 취재원 CRUD, 공유 공개 열람
```

### 엑셀 다운로드 제한
- 역할별 최대 행수 제한 (기자 100행 / 데스크 500행 / 슈퍼관리자 2000행)
- 1일 횟수 제한 (기자 3회 / 데스크 10회)
- 파일 메타데이터에 다운로드 기록 삽입

## 7. 도입 효과 (기대)

### 단기 효과
- **취재원 중복 등록 감소**: 등록 전 공유 목록 확인 습관화
- **인수인계 비용 절감**: 퇴사·이동 시 취재원 정보 자동 인계
- **부서 간 협업 강화**: "저 취재원 아는 분?" 도움 요청 시스템으로 정보 공유 활성화

### 중장기 효과
- **조직 취재원 DB 축적**: 수년간 축적된 기자들의 취재원 네트워크가 회사 자산으로 전환
- **신입 기자 온보딩 단축**: 선배 기자들이 등록한 취재원 정보를 바로 활용 가능
- **AI 고도화 기반 마련**: 누적된 취재원 데이터를 바탕으로 향후 AI 추천·분석 기능 확장 가능
