## 1. 과제 배경

### 왜 이 과제를 선택했는가

언론사에서 **취재원(Source)** 은 기사의 가장 핵심 자산이다. 그러나 이 자산은 대부분 기자 개인의 스마트폰 주소록과 메모장에 분산되어 있어, 조직의 공유 자산이 되지 못하고 있었다.

**실제 문제 상황:**

| 문제 | 결과 |
|------|------|
| 기자 개인 디바이스에 취재원 정보 보관 | 퇴사 시 수백 명의 취재원 정보가 조직에서 소멸 |
| 부서 내 정보 공유 불가 | 여러 기자가 동일 취재원에 중복 접촉 → 취재원 피로 유발 |
| 민감 정보 보호 장치 없음 | 파일 공유만으로 취재원 성향·비공개 발언 무단 유출 가능 |
| 관계망 파악 불가 | 탐사 기획 시 인맥 연결 고리를 활용할 수 없음 |
| 연락 이력 미기록 | 팔로업 누락, 후속 취재 단절 반복 |

이 문제들을 해결하기 위해 **"기자의 취재원 인맥을 조직의 영구 자산으로 전환"** 하는 시스템을 직접 설계·구현했다.

## 2. 결과물

### 2.1 시스템 개요

**취재원 관리 플랫폼(Source Manager)** — Next.js 16 + Supabase(PostgreSQL) 기반 풀스택 웹 애플리케이션

> 배포 환경: Vercel | 개발 언어: TypeScript | DB: Supabase PostgreSQL + RLS

---

### 2.2 구현된 핵심 기능 8가지

#### ① 취재원 DB 등록·관리
- 이름·소속·직책·연락처·학력·출신지역·SNS 등 20개 이상 필드
- **온더레코드 동의 상태 배지** (✅온더레코드 / 🟡백그라운드 / 🔴오프더레코드) — 인용 가능 여부 즉시 파악
- 완성도 점수 자동 계산 (기본정보 20 + 연락처 20 + 학력 20 + 정보 노트 40 = 100점)

#### ② 역할 기반 접근 제어 (7단계)
```
슈퍼관리자 → 편집인 → 국장 → 부국장 → 부장(소속부서) → 차장 → 기자
```
- 민감 정보: 차장 이상 자동 열람 / 기자는 데스크 승인 후 30일 한정 접근
- 부장: 소속 부서 정보보고만 열람 (타부서 차단)
- DB 레벨 Row Level Security로 API 우회 원천 차단

#### ③ 협업 정보 노트 시스템
- 여러 기자가 한 취재원에 공동으로 정보 추가
- **공개 노트** (전 기자 열람) / **민감 노트** (차장 이상만) 선택
- 정보 기여 시 +10pt 포인트 자동 지급

#### ④ 연락 이력 + 팔로업 관리
- 전화·문자·이메일·대면·기타 방법별 이력 타임라인
- **다음 팔로업 날짜** 설정 → 카드 상단에 배너 자동 표시 (3일 이내 녹색 강조)

#### ⑤ 인맥 네트워크 그래프
- 동료·대학동문·고교동문·고시동기·동향 등 자동 관계 감지
- **2촌 관계 검색**: 검색어 일치 취재원 + 그 연결 취재원까지 확장 탐색
- D3 Force 기반 인터랙티브 시각화

#### ⑥ 정보보고 시스템
- 취재 정보를 구조화 보고서로 작성 (단독/인터뷰/기획/일반 카테고리)
- 가시성 4단계: 전체공개 / 데스크이상 / 팀공개 / 작성자만
- 취재원 연동, 태그 검색, 부서별 접근 제어

#### ⑦ 포인트 시스템
- 취재원 등록·정보 추가·활용 기여도를 포인트로 정량화
- 전사 랭킹 및 기여 동기 부여

#### ⑧ 보안 및 감사 체계
- **5계층 보안**: JWT 인증 → 서버 세션 → RLS → API 역할 검증 → UI 조건 렌더링
- **제로-폭 워터마크**: 복사된 텍스트에 사용자 ID를 비가시 문자로 인코딩 → 유출자 추적
- **전체 감사 로그**: 열람·수정·민감 정보 접근·내보내기 이력 자동 기록
- 열람 승인 만료 (30일), VPN 접속 감지

## 3. 기대효과

### 3.1 업무 효율성 변화

| 지표 | 도입 전 | 도입 후 | 변화 |
|------|---------|---------|------|
| 취재원 검색 시간 | 5~20분 (동료 문의) | 1초 이내 | **-99%** |
| 퇴사자 정보 보존율 | 0% | 100% | **+100%p** |
| 중복 취재원 접촉 | 주 1~3건 | 거의 없음 | 구조적 방지 |
| 신입 취재원 파악 기간 | 6~12개월 | 입사 즉시 | **즉시 활용** |
| 팔로업 누락 | 빈번 | 배너 자동 알림 | 제로화 목표 |

### 3.2 정보 자산 가치화

- **조직 자산화**: 기자 개인에게 종속되어 있던 취재원 네트워크가 조직의 영구 DB로 전환
- **인맥 시너지**: 기자 개인이 파악 불가능했던 2~3촌 연결 고리 발견 → 탐사 기획 고도화
- **히스토리 누적**: 수년치 접촉 이력이 검색 가능한 형태로 조직에 축적

### 3.3 보안 수준 향상

- **민감 정보 노출 사고 차단**: 역할 기반 접근 제어로 구조적 방지
- **내부자 위협 억제**: 감사 로그 + 워터마크로 유출 시 추적 가능
- **취재원 신원 보호 강화**: 무단 열람 불가, 접근 이력 100% 기록

### 3.4 협업 문화 변화

```
[기존] "그 분야 취재원 아는 기자 있어?" → 단체 카톡 → 아무도 모름
[이후] 시스템 검색 → 2촌 탐색 → 연결 기자 즉시 파악 → 협력 취재
```

조직 내 정보 사일로가 해체되고, 선배 기자의 취재 노하우가 조직 전체로 전파되는 **지식 공유 문화**로의 전환이 기대된다.
